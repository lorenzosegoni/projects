Une approche par algorithme EM sur le locus de l’haptoglobine

Auteur·rice

Lorenzo Segoni

Date de publication

16 juillet 2026

Présentation

Dans une population consanguine, le coefficient de consanguinité \(f\) est la probabilité que deux gènes d’une personne aléatoire à un certain locus soient tous les deux des copies du même gène ancestral.

Nous observons les données suivantes sur les génotypes du gène haptoglobine de \(2048\) personnes dans le nord-est du Brésil.

Le locus de l’haptoglobine a trois allèles codominants, \(G1\), \(G2\) et \(G3\).

Génotype Nombre observé
\(G1/G1\) \(128\)
\(G1/G2\) \(196\)
\(G1/G3\) \(429\)
\(G2/G2\) \(173\)
\(G2/G3\) \(513\)
\(G3/G3\) \(609\)

Nous supposerons que cette population est à l’équilibre de Hardy-Weinberg. L’objectif principal est de déterminer s’il y a présence de consanguinité dans la population de laquelle est tiré cet échantillon. Nous en profiterons également pour estimer les fréquences alléliques \(p_1\), \(p_2\), \(p_3\) des allèles respectifs \(G1\), \(G2\), \(G3\) dans cette population.

1 Diagnostic préliminaire : mise en évidence d’un excès d’homozygotes

Avant toute modélisation fine, on cherche un indice rapide de consanguinité. La consanguinité augmente mécaniquement la proportion d’homozygotes : il suffit donc de comparer les effectifs homozygotes observés à ceux que prédirait un modèle sans consanguinité, où les proportions génotypiques sont purement de Hardy–Weinberg.

Code 
# Effectifs observés pour chaque génotype
G1_G1 = 128
G1_G2 = 196
G1_G3 = 429
G2_G2 = 173
G2_G3 = 513
G3_G3 = 609

# Taille totale de l'échantillon
n = sum( G1_G1 , G1_G2 , G1_G3 , G2_G2 , G2_G3 , G3_G3 )

# Comptage des allèles
f_G1 = 2 * G1_G1 + G1_G2 + G1_G3 
f_G2 = G1_G2 + 2 * G2_G2 + G2_G3 
f_G3 = G1_G3 + G2_G3 + 2 * G3_G3

# Estimation des fréquences alléliques
p_G1 = f_G1 / (2 * n)
p_G2 = f_G2 / (2 * n)
p_G3 = f_G3 / (2 * n)

# Probabilités génotypiques attendues sous Hardy-Weinberg
P_G1_G1 = p_G1 ** 2
P_G1_G2 = 2 * p_G1 * p_G2
P_G1_G3 = 2 * p_G1 * p_G3
P_G2_G2 = p_G2 ** 2
P_G2_G3 = 2 * p_G2 * p_G3
P_G3_G3 = p_G3 ** 2

# Effectifs attendus sous Hardy-Weinberg
G1_G1_HW = P_G1_G1 * n
G1_G2_HW = P_G1_G2 * n
G1_G3_HW = P_G1_G3 * n
G2_G2_HW = P_G2_G2 * n
G2_G3_HW = P_G2_G3 * n
G3_G3_HW = P_G3_G3 * n

# Comparaison du nombre total d'homozygotes
Homozygotes = G1_G1 + G2_G2 + G3_G3
Homozygotes_HW = G1_G1_HW + G2_G2_HW + G3_G3_HW

Un test du \(\chi^2\) d’ajustement quantifie l’écart global entre effectifs observés et attendus. Le modèle sans consanguinité ayant consommé deux degrés de liberté pour estimer les fréquences alléliques (trois fréquences liées par \(\sum p_i = 1\)), il reste \(6 - 1 - 2 = 3\) degrés de liberté.

Code 
A <- c(G1_G1 , G1_G2 , G1_G3 , G2_G2 , G2_G3 , G3_G3 )
B <- c(G1_G1_HW, G1_G2_HW, G1_G3_HW, G2_G2_HW,G2_G3_HW, G3_G3_HW)

# test X^2 
X_2 <- sum((A - B)^2 / B) # calcule statistique

Code 
p_value = 1 - pchisq(X_2, df = 3)

La statistique conduit à une \(p\)-valeur de l’ordre de \(1{,}3\times 10^{-7}\), très inférieure à \(0{,}05\). Le modèle d’équilibre de Hardy–Weinberg sans consanguinité est donc rejeté sans ambiguïté. Ce diagnostic ne quantifie pas encore \(f\) ; il justifie de passer à un modèle qui intègre explicitement la consanguinité.

2 Modélisation probabiliste et vraisemblances

Pour estimer \(f\), il faut un modèle qui relie ce paramètre aux probabilités génotypiques. L’idée-clé est de conditionner sur l’identité par descendance des deux allèles d’un individu.

Code 
# Initialisation
f <- 0.1
p1 <- p_G1
p2 <- p_G2
p3 <- p_G3
n11  <- 128 
n12 <- 196
n13 <- 429
n22  <- 173
n23 <- 513
n33 <- 609
n    <- sum (n11 , n12 , n13 , n22 , n23 , n33)

prob_homozygote <- function(f, p) {
  f * p + (1 - f) * p^2
}

prob_heterozygote <- function(f, p_i, p_j) {
  2 * (1 - f) * p_i * p_j
}

# Probabilités homozygotes
P11 <- prob_homozygote(f, p1)
P22 <- prob_homozygote(f, p2)
P33 <- prob_homozygote(f, p3)

# Probabilités hétérozygotes
P12 <- prob_heterozygote(f, p1, p2)
P13 <- prob_heterozygote(f, p1, p3)
P23 <- prob_heterozygote(f, p2, p3)

P_homozygote <- P11 + P22 + P33
P_heterozygote <- P12 + P13 + P23

# Log-vraisemblance observées

logv_obs <- function(f,p1,p2,p3, n11 = 128, n12 = 196, n13 = 429,n22 = 173, n23 = 513, n33 = 609) {
  ll <- n11 * log(prob_homozygote(f, p1)) +
    n22 * log(prob_homozygote(f, p2)) +
    n33 * log(prob_homozygote(f, p3)) +
    n12 * log(prob_heterozygote(f, p1, p2)) +
    n13 * log(prob_heterozygote(f, p1, p3)) +
    n23 * log(prob_heterozygote(f, p2, p3))
  
  return(ll)
}

# Log-vraisemblance complètes
logv_complete <- function(f,p1,p2,p3, n11_C, n22_C, n33_C , n11 = 128, n12 = 196, n13 = 429,n22 = 173, n23 = 513, n33 = 609) {
  ll <- n11_C * log(f * p1) +
    n22_C * log(f * p2) +
    n33_C * log(f * p3) +
    (n11 - n11_C) * log( (1 - f) * (p1 ** 2 ) ) +
    (n22 - n22_C) * log( (1 - f) * (p2 ** 2 ) ) +
    (n33 - n33_C) * log( (1 - f) * (p3 ** 2 ) ) +
    n12 * log( 2 * (1-f) * p1 * p2 ) +
    n13 * log( 2 * (1-f) * p1 * p3 ) +
    n23 * log( 2 * (1-f) * p2 * p3 ) 
  
  return(ll)
}

3 Estimation par l’algorithme EM

L’algorithme EM alterne l’espérance des données manquantes (étape E) et la maximisation de la vraisemblance complète espérée (étape M), en garantissant une croissance monotone de \(\ell_{\text{obs}}\) à chaque itération.

Code 
# Observation
obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

# EM
EM <- function( genotype , theta_init = c(f = 0.1, p1 = 0.33, p2 = 0.33, p3 = 0.34), eps = 1e-8,  max_iter = 2000){
  
  # Effectifs
  n11 <- genotype["n11"] 
  n12 <- genotype["n12"] 
  n13 <- genotype["n13"]
  n22 <- genotype["n22"]
  n23 <- genotype["n23"]
  n33 <- genotype["n33"]
  
  n <- sum(genotype)
  
  # Initialisation
  f  <- theta_init["f"]
  p1 <- theta_init["p1"]
  p2 <- theta_init["p2"]
  p3 <- theta_init["p3"]
  
  theta_old <- c(f, p1, p2, p3)
  
  # Matrice des itératios 
  trace <- matrix(NA, nrow = max_iter + 1, ncol = 4)
  colnames(trace) <- c("f", "p1", "p2", "p3")
  trace[1, ] <- theta_old
  
  iter <- 1
  
  while (iter <= max_iter) {
    
    # ÉTAPE E
    denom1 <- f * p1 + (1 - f) * p1^2
    denom2 <- f * p2 + (1 - f) * p2^2
    denom3 <- f * p3 + (1 - f) * p3^2
    
    gamma1 <- f * p1 / denom1
    gamma2 <- f * p2 / denom2
    gamma3 <- f * p3 / denom3
    
    n11_C <- n11 * gamma1
    n22_C <- n22 * gamma2
    n33_C <- n33 * gamma3
    
    n11_barC <- n11 - n11_C
    n22_barC <- n22 - n22_C
    n33_barC <- n33 - n33_C
    
    
    # ÉTAPE M
    f_new <- (n11_C + n22_C + n33_C) / n
    
    M1 <- n11_C + 2 * n11_barC + (n12 + n13)
    M2 <- n22_C + 2 * n22_barC + (n12 + n23)
    M3 <- n33_C + 2 * n33_barC + (n13 + n23)
    
    M_sum <- M1 + M2 + M3
    
    p1_new <- M1 / M_sum
    p2_new <- M2 / M_sum
    p3_new <- M3 / M_sum
    
    theta_new <- c(f_new, p1_new, p2_new, p3_new)
    
    trace[iter + 1, ] <- theta_new
    
    
    # CRITÈRE DE CONVERGENCE
    if (sqrt(sum((theta_new - theta_old)^2)) < eps) {
      trace <- trace[1:(iter + 1), , drop = FALSE]
      return(as.data.frame(trace))
    }
    
    # Mise à jour des données
    theta_old <- theta_new
    f  <- f_new
    p1 <- p1_new
    p2 <- p2_new
    p3 <- p3_new
    
    iter <- iter + 1
  }
  
  as.data.frame(trace)
}

4 Quantification de l’incertitude par bootstrap

L’estimation ponctuelle ne dit rien de sa précision. Faute d’expression close simple pour la matrice de covariance des estimateurs EM, on l’approche par bootstrap : on régénère un grand nombre d’échantillons, on ré-estime \(\theta\) sur chacun, et l’on prend la covariance empirique des estimations.

On met en œuvre les deux variantes classiques afin de les comparer :

  • bootstrap empirique (non paramétrique) : on rééchantillonne les \(n\) individus avec remise à partir des proportions génotypiques observées ;
  • bootstrap paramétrique : on régénère les effectifs selon une multinomiale dont les probabilités sont celles du modèle évalué en \(\hat\theta\).
Code 
obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

bootstrap_empirique <- function( genotype , B = 10000 , theta_init = c(f = 0.1, p1 = 0.33, p2 = 0.33, p3 = 0.34), eps = 1e-8) {
  
  # Initialisation
  n <- sum(genotype)
  geno_names <- c("n11", "n12", "n13", "n22", "n23", "n33")
  probs <- genotype / n
  
  # Matrice pour stocker les estimateurs bootstrap
  theta_mat <- matrix(NA, nrow = B, ncol = 4)
  colnames(theta_mat) <- c("f", "p1", "p2", "p3")
  
  for (b in 1:B) {
    # Rééchantillonnage des génotypes
    samp <- sample(geno_names, size = n, replace = TRUE, prob = probs)
    
    # Reconstruction du vecteur de comptages bootstrap
    genotype_b <- table(factor(samp, levels = geno_names))
    genotype_b <- as.numeric(genotype_b)
    names(genotype_b) <- geno_names
    
    # Exécution EM
    trace_b <- EM(genotype_b, theta_init = theta_init, eps = eps)
    theta_hat_b <- as.numeric(tail(trace_b, 1))
    
    # Stockage
    theta_mat[b, ] <- theta_hat_b
  }
  
  list(theta_mat = theta_mat, cov_hat   = cov(theta_mat))
}

La matrice de covariance fournit les écarts-types des quatre paramètres. On obtient une précision de l’ordre de \(\pm 0{,}017\) sur \(\hat f\) et de \(\pm 0{,}007\) sur chaque fréquence allélique — le coefficient de consanguinité, \(\hat f \approx 0{,}091\), se détache donc largement de zéro au regard de sa propre incertitude.

Quant à la comparaison des deux méthodes (le bonus), les écarts-types empiriques et paramétriques coïncident à la troisième décimale près. Ce n’est pas un hasard : le bootstrap non paramétrique rééchantillonne selon la multinomiale empirique, tandis que le paramétrique rééchantillonne selon la multinomiale ajustée ; or le modèle épouse très bien les données (\(f\) et les \(p_i\) sont estimés sur les mêmes six proportions), si bien que les deux lois de rééchantillonnage sont presque identiques. Les deux approches sont donc ici interchangeables ; elles divergeraient si le modèle était mal spécifié.

5 Significativité du coefficient de consanguinité

Le diagnostic initial rejetait Hardy–Weinberg, mais on veut maintenant tester directement l’hypothèse \(H_0 : f = 0\) dans le cadre du modèle avec consanguinité, contre \(H_1 : f > 0\). On utilise le test du rapport de vraisemblance.

Code 
obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

# Estimation EM
theta_hat <- tail(EM(obs, theta_init = c(f=0.1, p1=0.33, p2=0.33, p3=0.34), eps = 1e-8, max_iter = 2000), n = 1)
f_hat <- theta_hat$f
p1_hat <- theta_hat$p1
p2_hat <- theta_hat$p2
p3_hat <- theta_hat$p3

# Estimation avec f = 0
n <- sum(obs)
p1_tilde <- (2*obs["n11"] + obs["n12"] + obs["n13"]) / (2*n)
p2_tilde <- (2*obs["n22"] + obs["n12"] + obs["n23"]) / (2*n)
p3_tilde <- (2*obs["n33"] + obs["n13"] + obs["n23"]) / (2*n)

# logvraisemblance observé
l1 <- logv_obs(f_hat,  p1_hat,   p2_hat,   p3_hat)
l0 <- logv_obs(0,      p1_tilde, p2_tilde, p3_tilde)

# Test de vraisemblance
Lambda   <- 2 * (l1 - l0)
p_valeur <- pchisq(Lambda, df = 1, lower.tail = FALSE)

cat("Lambda =",   round(Lambda,   4), "\n")

La statistique vaut \(\Lambda \approx 32{,}4\), à comparer au seuil \(\chi^2_{1;\,0{,}95} \approx 3{,}84\). La consanguinité est hautement significative dans cette population : l’excès d’homozygotie repéré au départ n’est pas imputable au hasard d’échantillonnage, mais bien à une valeur positive de \(f\).

6 Structure de dépendance entre \(f\) et \(p_1\)

Une dernière question porte sur la géométrie de l’estimation : le coefficient de consanguinité est-il estimé indépendamment des fréquences alléliques, ou bien existe-t-il une confusion entre eux ? On lit la réponse dans la matrice de corrélation des estimateurs bootstrap.

La corrélation estimée entre \(\hat f\) et \(\hat p_1\) est proche de zéro (de l’ordre de \(0{,}02\)–\(0{,}04\) selon la graine et la variante de bootstrap). Les deux paramètres sont donc, à toutes fins pratiques, estimés indépendamment : le coefficient de consanguinité n’est pas confondu avec la fréquence de l’allèle \(G1\). Autrement dit, l’information sur la consanguinité provient de l’excès global d’homozygotes, quantité largement orthogonale à la composition allélique du pool.

Conclusion

La population du Nord-Est brésilien étudiée présente une consanguinité modérée mais statistiquement indéniable, de l’ordre de \(9\,\%\), au locus de l’haptoglobine.

# Présentation {.unnumbered}

Dans une population consanguine, le coefficient de consanguinité $f$ est la probabilité que deux gènes d'une personne aléatoire à un certain locus soient tous les deux des copies du même gène ancestral.

Nous observons les données suivantes sur les génotypes du gène haptoglobine de $2048$ personnes dans le nord-est du Brésil.

Le locus de l'haptoglobine a trois allèles codominants, $G1$, $G2$ et $G3$.

| Génotype | Nombre observé |
|----------|----------------|
| $G1/G1$  | $128$          |
| $G1/G2$  | $196$          |
| $G1/G3$  | $429$          |
| $G2/G2$  | $173$          |
| $G2/G3$  | $513$          |
| $G3/G3$  | $609$          |

Nous supposerons que cette population est à l'équilibre de Hardy-Weinberg. L'objectif principal est de déterminer s'il y a présence de consanguinité dans la population de laquelle est tiré cet échantillon. Nous en profiterons également pour estimer les fréquences alléliques $p_1$, $p_2$, $p_3$ des allèles respectifs $G1$, $G2$, $G3$ dans cette population.

# Diagnostic préliminaire : mise en évidence d'un excès d'homozygotes

Avant toute modélisation fine, on cherche un indice rapide de consanguinité. La consanguinité augmente mécaniquement la proportion d'homozygotes : il suffit donc de comparer les effectifs homozygotes observés à ceux que prédirait un modèle **sans** consanguinité, où les proportions génotypiques sont purement de Hardy–Weinberg.

```{r}
#| label: diagnostic

# Effectifs observés pour chaque génotype
G1_G1 = 128
G1_G2 = 196
G1_G3 = 429
G2_G2 = 173
G2_G3 = 513
G3_G3 = 609

# Taille totale de l'échantillon
n = sum( G1_G1 , G1_G2 , G1_G3 , G2_G2 , G2_G3 , G3_G3 )

# Comptage des allèles
f_G1 = 2 * G1_G1 + G1_G2 + G1_G3 
f_G2 = G1_G2 + 2 * G2_G2 + G2_G3 
f_G3 = G1_G3 + G2_G3 + 2 * G3_G3

# Estimation des fréquences alléliques
p_G1 = f_G1 / (2 * n)
p_G2 = f_G2 / (2 * n)
p_G3 = f_G3 / (2 * n)

# Probabilités génotypiques attendues sous Hardy-Weinberg
P_G1_G1 = p_G1 ** 2
P_G1_G2 = 2 * p_G1 * p_G2
P_G1_G3 = 2 * p_G1 * p_G3
P_G2_G2 = p_G2 ** 2
P_G2_G3 = 2 * p_G2 * p_G3
P_G3_G3 = p_G3 ** 2

# Effectifs attendus sous Hardy-Weinberg
G1_G1_HW = P_G1_G1 * n
G1_G2_HW = P_G1_G2 * n
G1_G3_HW = P_G1_G3 * n
G2_G2_HW = P_G2_G2 * n
G2_G3_HW = P_G2_G3 * n
G3_G3_HW = P_G3_G3 * n

# Comparaison du nombre total d'homozygotes
Homozygotes = G1_G1 + G2_G2 + G3_G3
Homozygotes_HW = G1_G1_HW + G2_G2_HW + G3_G3_HW
Homozygotes
Homozygotes_HW
```

Un test du $\chi^2$ d'ajustement quantifie l'écart global entre effectifs observés et attendus. Le modèle sans consanguinité ayant consommé deux degrés de liberté pour estimer les fréquences alléliques (trois fréquences liées par $\sum p_i = 1$), il reste $6 - 1 - 2 = 3$ degrés de liberté.

```{r}
#| label: chi2

A <- c(G1_G1 , G1_G2 , G1_G3 , G2_G2 , G2_G3 , G3_G3 )
B <- c(G1_G1_HW, G1_G2_HW, G1_G3_HW, G2_G2_HW,G2_G3_HW, G3_G3_HW)

# test X^2 
X_2 <- sum((A - B)^2 / B) # calcule statistique
X_2
p_value = 1 - pchisq(X_2, df = 3) 
p_value
```

La statistique conduit à une $p$-valeur de l'ordre de $1{,}3\times 10^{-7}$, très inférieure à $0{,}05$. Le modèle d'équilibre de Hardy–Weinberg **sans** consanguinité est donc rejeté sans ambiguïté. Ce diagnostic ne quantifie pas encore $f$ ; il justifie de passer à un modèle qui intègre explicitement la consanguinité.

# Modélisation probabiliste et vraisemblances

Pour estimer $f$, il faut un modèle qui relie ce paramètre aux probabilités génotypiques. L'idée-clé est de **conditionner sur l'identité par descendance** des deux allèles d'un individu.

```{r}
#| label: vraisemblances

# Initialisation
f <- 0.1
p1 <- p_G1
p2 <- p_G2
p3 <- p_G3
n11  <- 128 
n12 <- 196
n13 <- 429
n22  <- 173
n23 <- 513
n33 <- 609
n    <- sum (n11 , n12 , n13 , n22 , n23 , n33)

prob_homozygote <- function(f, p) {
  f * p + (1 - f) * p^2
}

prob_heterozygote <- function(f, p_i, p_j) {
  2 * (1 - f) * p_i * p_j
}

# Probabilités homozygotes
P11 <- prob_homozygote(f, p1)
P22 <- prob_homozygote(f, p2)
P33 <- prob_homozygote(f, p3)

# Probabilités hétérozygotes
P12 <- prob_heterozygote(f, p1, p2)
P13 <- prob_heterozygote(f, p1, p3)
P23 <- prob_heterozygote(f, p2, p3)

P_homozygote <- P11 + P22 + P33
P_heterozygote <- P12 + P13 + P23

# Log-vraisemblance observées

logv_obs <- function(f,p1,p2,p3, n11 = 128, n12 = 196, n13 = 429,n22 = 173, n23 = 513, n33 = 609) {
  ll <- n11 * log(prob_homozygote(f, p1)) +
    n22 * log(prob_homozygote(f, p2)) +
    n33 * log(prob_homozygote(f, p3)) +
    n12 * log(prob_heterozygote(f, p1, p2)) +
    n13 * log(prob_heterozygote(f, p1, p3)) +
    n23 * log(prob_heterozygote(f, p2, p3))
  
  return(ll)
}

# Log-vraisemblance complètes
logv_complete <- function(f,p1,p2,p3, n11_C, n22_C, n33_C , n11 = 128, n12 = 196, n13 = 429,n22 = 173, n23 = 513, n33 = 609) {
  ll <- n11_C * log(f * p1) +
    n22_C * log(f * p2) +
    n33_C * log(f * p3) +
    (n11 - n11_C) * log( (1 - f) * (p1 ** 2 ) ) +
    (n22 - n22_C) * log( (1 - f) * (p2 ** 2 ) ) +
    (n33 - n33_C) * log( (1 - f) * (p3 ** 2 ) ) +
    n12 * log( 2 * (1-f) * p1 * p2 ) +
    n13 * log( 2 * (1-f) * p1 * p3 ) +
    n23 * log( 2 * (1-f) * p2 * p3 ) 
  
  return(ll)
}
```

# Estimation par l'algorithme EM

L'algorithme EM alterne l'espérance des données manquantes (étape E) et la maximisation de la vraisemblance complète espérée (étape M), en garantissant une croissance monotone de $\ell_{\text{obs}}$ à chaque itération.

```{r}
#| label: em-fonction

# Observation
obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

# EM
EM <- function( genotype , theta_init = c(f = 0.1, p1 = 0.33, p2 = 0.33, p3 = 0.34), eps = 1e-8,  max_iter = 2000){
  
  # Effectifs
  n11 <- genotype["n11"] 
  n12 <- genotype["n12"] 
  n13 <- genotype["n13"]
  n22 <- genotype["n22"]
  n23 <- genotype["n23"]
  n33 <- genotype["n33"]
  
  n <- sum(genotype)
  
  # Initialisation
  f  <- theta_init["f"]
  p1 <- theta_init["p1"]
  p2 <- theta_init["p2"]
  p3 <- theta_init["p3"]
  
  theta_old <- c(f, p1, p2, p3)
  
  # Matrice des itératios 
  trace <- matrix(NA, nrow = max_iter + 1, ncol = 4)
  colnames(trace) <- c("f", "p1", "p2", "p3")
  trace[1, ] <- theta_old
  
  iter <- 1
  
  while (iter <= max_iter) {
    
    # ÉTAPE E
    denom1 <- f * p1 + (1 - f) * p1^2
    denom2 <- f * p2 + (1 - f) * p2^2
    denom3 <- f * p3 + (1 - f) * p3^2
    
    gamma1 <- f * p1 / denom1
    gamma2 <- f * p2 / denom2
    gamma3 <- f * p3 / denom3
    
    n11_C <- n11 * gamma1
    n22_C <- n22 * gamma2
    n33_C <- n33 * gamma3
    
    n11_barC <- n11 - n11_C
    n22_barC <- n22 - n22_C
    n33_barC <- n33 - n33_C
    
    
    # ÉTAPE M
    f_new <- (n11_C + n22_C + n33_C) / n
    
    M1 <- n11_C + 2 * n11_barC + (n12 + n13)
    M2 <- n22_C + 2 * n22_barC + (n12 + n23)
    M3 <- n33_C + 2 * n33_barC + (n13 + n23)
    
    M_sum <- M1 + M2 + M3
    
    p1_new <- M1 / M_sum
    p2_new <- M2 / M_sum
    p3_new <- M3 / M_sum
    
    theta_new <- c(f_new, p1_new, p2_new, p3_new)
    
    trace[iter + 1, ] <- theta_new
    
    
    # CRITÈRE DE CONVERGENCE
    if (sqrt(sum((theta_new - theta_old)^2)) < eps) {
      trace <- trace[1:(iter + 1), , drop = FALSE]
      return(as.data.frame(trace))
    }
    
    # Mise à jour des données
    theta_old <- theta_new
    f  <- f_new
    p1 <- p1_new
    p2 <- p2_new
    p3 <- p3_new
    
    iter <- iter + 1
  }
  
  as.data.frame(trace)
}
```

```{r}
#| label: em-resultat

trace_EM   <- EM(obs, eps = 1e-10, max_iter = 5000)
theta_hat  <- trace_EM[nrow(trace_EM), ]
round(theta_hat, 5)
```

L'algorithme converge vers $\hat f \approx 0{,}091$, avec des fréquences alléliques $\hat p_1 \approx 0{,}215$, $\hat p_2 \approx 0{,}257$ et $\hat p_3 \approx 0{,}528$. Les fréquences restent très proches de l'estimation naïve par comptage de gènes — attendu, puisque la consanguinité redistribue les génotypes sans modifier le pool allélique — tandis que $f \approx 0{,}09$ chiffre pour la première fois l'intensité de la consanguinité suggérée par le diagnostic.

# Quantification de l'incertitude par bootstrap

L'estimation ponctuelle ne dit rien de sa précision. Faute d'expression close simple pour la matrice de covariance des estimateurs EM, on l'approche par **bootstrap** : on régénère un grand nombre d'échantillons, on ré-estime $\theta$ sur chacun, et l'on prend la covariance empirique des estimations.

On met en œuvre les deux variantes classiques afin de les comparer :

- **bootstrap empirique (non paramétrique)** : on rééchantillonne les $n$ individus avec remise à partir des proportions génotypiques observées ;
- **bootstrap paramétrique** : on régénère les effectifs selon une multinomiale dont les probabilités sont celles du modèle évalué en $\hat\theta$.

```{r}
#| label: bootstrap-fonctions
#| cache: true

obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

bootstrap_empirique <- function( genotype , B = 10000 , theta_init = c(f = 0.1, p1 = 0.33, p2 = 0.33, p3 = 0.34), eps = 1e-8) {
  
  # Initialisation
  n <- sum(genotype)
  geno_names <- c("n11", "n12", "n13", "n22", "n23", "n33")
  probs <- genotype / n
  
  # Matrice pour stocker les estimateurs bootstrap
  theta_mat <- matrix(NA, nrow = B, ncol = 4)
  colnames(theta_mat) <- c("f", "p1", "p2", "p3")
  
  for (b in 1:B) {
    # Rééchantillonnage des génotypes
    samp <- sample(geno_names, size = n, replace = TRUE, prob = probs)
    
    # Reconstruction du vecteur de comptages bootstrap
    genotype_b <- table(factor(samp, levels = geno_names))
    genotype_b <- as.numeric(genotype_b)
    names(genotype_b) <- geno_names
    
    # Exécution EM
    trace_b <- EM(genotype_b, theta_init = theta_init, eps = eps)
    theta_hat_b <- as.numeric(tail(trace_b, 1))
    
    # Stockage
    theta_mat[b, ] <- theta_hat_b
  }
  
  list(theta_mat = theta_mat, cov_hat   = cov(theta_mat))
}

#Bonus
resulat0 <- EM(obs, theta_init = c(f = 0.1, p1 = 0.33, p2 = 0.33, p3 = 0.34), eps = 1e-8, max_iter = 2000)
resulat0_fin <- resulat0[nrow(resulat0), ]

prob_6types_G <- c(

  "22" = f2*pB2 + (1-f2)*pB2^2,
  "23" = (1-f2)*2*pB2*pC2,
  "33" = f2*pC2 + (1-f2)*pC2^2
)
sum(prob_6types_G) #[1] 1

# B <- 10000
Echantillons2 <- matrix(0, nrow = B, ncol = 4)
colnames(Echantillons2) <- c("pA2","pB2","pC2","f2")

for(b in 1:B){
  echantillon_tab2 <- as.integer(rmultinom(1, size = n, prob = prob_6types_G))
  genotype_b <- c(
    n11 = echantillon_tab2[1],
    n12 = echantillon_tab2[2],

  )
  sor2 <- EM(genotype_b, theta_init = c(f = f2, p1 = pA2, p2 = pB2, p3 = pC2), eps = 1e-8, max_iter = 2000)
  fin2 <- sor2[nrow(sor2), ]
  Echantillons2[b, ] <- c(

}
```

La matrice de covariance fournit les écarts-types des quatre paramètres. On obtient une précision de l'ordre de $\pm 0{,}017$ sur $\hat f$ et de $\pm 0{,}007$ sur chaque fréquence allélique — le coefficient de consanguinité, $\hat f \approx 0{,}091$, se détache donc largement de zéro au regard de sa propre incertitude.

Quant à la comparaison des deux méthodes (le **bonus**), les écarts-types empiriques et paramétriques coïncident à la troisième décimale près. Ce n'est pas un hasard : le bootstrap non paramétrique rééchantillonne selon la multinomiale *empirique*, tandis que le paramétrique rééchantillonne selon la multinomiale *ajustée* ; or le modèle épouse très bien les données ($f$ et les $p_i$ sont estimés sur les mêmes six proportions), si bien que les deux lois de rééchantillonnage sont presque identiques. Les deux approches sont donc ici interchangeables ; elles divergeraient si le modèle était mal spécifié.

# Significativité du coefficient de consanguinité

Le diagnostic initial rejetait Hardy–Weinberg, mais on veut maintenant tester directement l'hypothèse $H_0 : f = 0$ *dans le cadre du modèle avec consanguinité*, contre $H_1 : f > 0$. On utilise le **test du rapport de vraisemblance**.

```{r}
#| label: test-lrt

obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

# Estimation EM
theta_hat <- tail(EM(obs, theta_init = c(f=0.1, p1=0.33, p2=0.33, p3=0.34), eps = 1e-8, max_iter = 2000), n = 1)
f_hat <- theta_hat$f
p1_hat <- theta_hat$p1
p2_hat <- theta_hat$p2
p3_hat <- theta_hat$p3

# Estimation avec f = 0
n <- sum(obs)
p1_tilde <- (2*obs["n11"] + obs["n12"] + obs["n13"]) / (2*n)
p2_tilde <- (2*obs["n22"] + obs["n12"] + obs["n23"]) / (2*n)
p3_tilde <- (2*obs["n33"] + obs["n13"] + obs["n23"]) / (2*n)

# logvraisemblance observé
l1 <- logv_obs(f_hat,  p1_hat,   p2_hat,   p3_hat)
l0 <- logv_obs(0,      p1_tilde, p2_tilde, p3_tilde)

# Test de vraisemblance
Lambda   <- 2 * (l1 - l0)
p_valeur <- pchisq(Lambda, df = 1, lower.tail = FALSE)

cat("Lambda =",   round(Lambda,   4), "\n")
cat("p-valeur =", formatC(p_valeur, format = "e", digits = 3), "\n")
```

La statistique vaut $\Lambda \approx 32{,}4$, à comparer au seuil $\chi^2_{1;\,0{,}95} \approx 3{,}84$. La $p$-valeur associée, de l'ordre de $1{,}3\times 10^{-8}$, mène à un rejet franc de $H_0$. **La consanguinité est hautement significative** dans cette population : l'excès d'homozygotie repéré au départ n'est pas imputable au hasard d'échantillonnage, mais bien à une valeur positive de $f$.

# Structure de dépendance entre $f$ et $p_1$

Une dernière question porte sur la géométrie de l'estimation : le coefficient de consanguinité est-il estimé *indépendamment* des fréquences alléliques, ou bien existe-t-il une confusion entre eux ? On lit la réponse dans la matrice de corrélation des estimateurs bootstrap.

```{r}
#| label: correlation

obs <- c(n11 = 128, n12 = 196, n13 = 429, n22 = 173, n23 = 513, n33 = 609)

boot_emp <- bootstrap_empirique(obs, B = 10000)

# Correlation
cor_matrix <- cor(boot_emp$theta_mat)
cor_matrix
cor_f_p1   <- cor_matrix["f", "p1"]

cat("Corrélation estimée entre f et p1 :", round(cor_f_p1, 4), "\n")
```

La corrélation estimée entre $\hat f$ et $\hat p_1$ est proche de zéro (de l'ordre de $0{,}02$–$0{,}04$ selon la graine et la variante de bootstrap). Les deux paramètres sont donc, à toutes fins pratiques, **estimés indépendamment** : le coefficient de consanguinité n'est pas confondu avec la fréquence de l'allèle $G1$. Autrement dit, l'information sur la consanguinité provient de l'excès *global* d'homozygotes, quantité largement orthogonale à la composition allélique du pool.

# Conclusion {.unnumbered}

La population du Nord-Est brésilien étudiée présente une consanguinité modérée mais statistiquement indéniable, de l'ordre de $9\,\%$, au locus de l'haptoglobine.
